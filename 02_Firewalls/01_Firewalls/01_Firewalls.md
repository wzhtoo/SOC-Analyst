# Firewalls

### နိဒါန်း
Firewall ဆိုသည်မှာ ယုံကြည်ရသော ကွန်ယက် (ဥပမာ- ကိုယ်ပိုင်ကွန်ယက်) နှင့် မယုံကြည်ရသော ကွန်ယက်များ (ဥပမာ- အင်တာနက်) အကြားရှိ ဆက်သွယ်မှုများကို စောင့်ကြည့်ထိန်းချုပ်သည့် ကွန်ယက် လုံခြုံရေး စနစ်တစ်ခု ဖြစ်သည်။ ၎င်းသည် ကြိုတင်သတ်မှတ်ထားသော လုံခြုံရေး စည်းမျဉ်းများအရ ကွန်ယက် အသွားအလာ (Traffic) ကို ခွင့်ပြုခြင်း သို့မဟုတ် ပိတ်ပင်ခြင်း လုပ်ဆောင်သည်။

### Firewall ၏ အခြေခံ လုပ်ဆောင်ချက်
Firewall ၏ အဓိက လုပ်ဆောင်ချက်မှာ "Filtering" လုပ်ခြင်းဖြစ်သည်။ ထိုသို့ စစ်ထုတ်ရာတွင် အဓိက နည်းလမ်းသုံးမျိုး ရှိပါသည်။

1.  **Packet Filtering Firewall** - ၎င်းသည် အစောဆုံးနှင့် အခြေခံအကျဆုံး ဖြစ်သည်။ Data Packet တစ်ခု၏ ဦးခေါင်း (Header) ရှိ အချက်အလက်များ (Source IP, Destination IP, Protocol, Port နံပါတ်) ကို ကြည့်၍ သတ်မှတ်စည်းမျဉ်းများနှင့် ကိုက်ညီမှုရှိမရှိ စစ်ဆေးကာ ခွင့်ပြု သို့မဟုတ် ငြင်းပယ်သည်။

2.  **Stateful Inspection Firewall** - ၎င်းသည် Packet Filtering ထက် ပိုမိုတိုးတက်ပြီး စမတ်ကျသည်။ Packet တစ်ခုချင်းစီကို သီးသန့်ကြည့်ရုံသာမက၊ ဆက်သွယ်မှု တစ်ခုလုံး၏ "အခြေအနေ" ကို ခြေရာခံသည်။ ဥပမာ၊ ကိုယ်ပိုင်ကွန်ယက်မှ အင်တာနက်သို့ တောင်းဆိုမှုတစ်ခု ထွက်သွားပါက၊ ပြန်လာသော Data ကို အတည်ပြုထားသော တုံ့ပြန်မှုအဖြစ် အလိုအလျောက် ခွင့်ပြုပေးသည်။

3.  **Application-Level Gateway (Proxy Firewall)** - ၎င်းသည် အလွှာအလိုက် စစ်ဆေးခြင်းဖြစ်ပြီး အသုံးချပရိုဂရမ် (Application) အဆင့်အထိ စစ်ဆေးနိုင်သည်။ ၎င်းသည် ကွန်ယက်တွင်းရှိ ကွန်ပျူတာများနှင့် အပြင်ဘက်ကွန်ယက်ကြားတွင် ယာယီဆက်သွယ်မှု (Proxy) တစ်ခုအဖြစ် ဆောင်ရွက်ကာ ဆက်သွယ်မှုအားလုံးကို ၎င်းက ကိုယ်စားပြုဆောင်ရွက်သည်။ ထို့ကြောင့် အင်တာနက်ပေါ်ရှိ Server များသည် ကွန်ယက်အတွင်းရှိ ကွန်ပျူတာများ၏ တိုက်ရိုက်လိပ်စာကို မမြင်ရဘဲ Proxy Server ၏လိပ်စာကိုသာ မြင်ရသည်။ ၎င်းသည် စစ်ထုတ်မှုအပြင် ကြားခံ ကာကွယ်မှုလည်း ပေးသည်။

### Firewall ၏ အဓိက ရည်ရွယ်ချက်များ
*   **မလိုလားအပ်သော ဝင်ရောက်မှုကို တားဆီးခြင်း** - Hacker များ၊ Malware များ၏ စစ်တုရင် စမ်းသပ်မှုများကို ကာကွယ်ပေးသည်။
*   **အားနည်းချက် စစ်ဆေးရာနေရာကို လျှော့ချခြင်း** - ကွန်ယက်အတွင်းရှိ ဝန်ဆောင်မှုများ (ဥပမာ- File Sharing, Remote Desktop) ကို အင်တာနက်မှ တိုက်ရိုက်ဝင်ရောက်၍မရအောင် ပိတ်ထားနိုင်သည်။
*   **ကွန်ယက်အသွားအလာကို စောင့်ကြည့်ခြင်းနှင့် မှတ်တမ်းတင်ခြင်း** - Firewall များသည် ဖြတ်သန်းသွားလာသော Traffic အားလုံး၏ မှတ်တမ်း (Log) ကို သိမ်းဆည်းထားနိုင်သည်။ ထိုမှတ်တမ်းများကို ခွဲခြမ်းစိတ်ဖြာ၍ ထူးခြားသော သို့မဟုတ် အန္တရာယ်ရှိသော လှုပ်ရှားမှုများကို ရှာဖွေနိုင်သည်။
*   **ပေါ်လစီအရ ထိန်းချုပ်ခြင်း** - လုပ်ငန်းတွင်း လုံခြုံရေးပေါ်လစီများကို အကောင်အထည်ဖော်ရန် ကူညီပေးသည်။ ဥပမာ- အချို့ဝက်ဘ်ဆိုဒ်များကို လုပ်ငန်းချိန်အတွင်း ဝင်ရောက်ခွင့်ပိတ်ခြင်း။

### Firewall အမျိုးအစားများ (တပ်ဆင်ပုံနှင့် တည်နေရာအလိုက်)
1.  **Software Firewall** - ၎င်းကို ကွန်ပျူတာ တစ်လုံးချင်းစီတွင် ထည့်သွင်းအသုံးပြုသည်။ (ဥပမာ- Windows Defender Firewall, Third-Party Antivirus ပါ Firewall)။ ၎င်းသည် ထိုကွန်ပျူတာတစ်ခုတည်းကို ကာကွယ်ပေးသည်။
2.  **Hardware Firewall** - ၎င်းသည် ရုပ်ဝတ္ထု ပစ္စည်းတစ်ခုဖြစ်ပြီး၊ ကွန်ယက်၏ အဝင်ပိုင်းတွင် တပ်ဆင်ကာ ကွန်ယက်တစ်ခုလုံးကို ကာကွယ်ပေးသည်။ အိမ်သုံး Router အများစုတွင် အခြေခံ Hardware Firewall ပါရှိသည်။
3.  **Next-Generation Firewall (NGFW)** - ယနေ့ခေတ်တွင် ကျယ်ပြန့်စွာ အသုံးပြုနေသော Firewall အမျိုးအစားဖြစ်သည်။ ၎င်းသည် Stateful Inspection အပြင်၊
    *   **အန္တရာယ်ရှိသော Application များကို ခွဲခြားခြင်း** (ဥပမာ- BitTorrent, Skype)
    *   **Intrusion Prevention Systems (IPS)** - ကျူးကျော်မှုများကို အလိုအလျောက် ရပ်တန့်ခြင်း
    *   **Deep Packet Inspection (DPI)** - Packet ၏ Data အပိုင်း (Header သာမက Payload) ကို နက်ရှိုင်းစွာ စစ်ဆေးခြင်း
    စသည့် လုပ်ဆောင်ချက်များကို ပေါင်းစပ်ထည့်သွင်းထားသည်။

### Firewall ၏ ကန့်သတ်ချက်များ
Firewall သည် စွမ်းဆောင်ရည်မြင့်သော်လည်း၊ သူတစ်ခုတည်းဖြင့် လုံခြုံရေး အပြည့်အဝ မရနိုင်ပါ။
*   **အတွင်းမှ ခြိမ်းခြောက်မှု** - ကွန်ယက်အတွင်းမှ ကွန်ပျူတာတစ်လုံးသည် Malware ကူးစက်ခံရပြီး အပြင်ဘက်သို့ Data များ ပို့နေပါက၊ ထိုအသွားအလာသည် ခွင့်ပြုထားသော Traffic (ဥပမာ- HTTPS) အဖြစ် ရှိနေနိုင်သဖြင့် Firewall က မတားဆီးနိုင်ပါ။
*   **Encrypted Traffic** - ယနေ့ Web Traffic အများစုသည် SSL/TLS ဖြင့် လုံခြုံစွာ ချုပ်ကာ (Encrypt) ပို့သည်။ Firewall သည် ထို Traffic ကို စစ်ဆေးလိုပါက၊ ထိုလုံခြုံမှုကို ဖြေရှင်းပြီးမှ စစ်ဆေးရန် လိုအပ်သည် (ထိုသို့လုပ်ဆောင်နိုင်သော Firewall များလည်းရှိသည်)။
*   **အသုံးချပရိုဂရမ် အားနည်းချက်များ** - Firewall က ပုံမှန်ဝင်ပေါက်များကို ပိတ်ထားသော်လည်း၊ ခွင့်ပြုထားသော Application (ဥပမာ- Web Server) တွင် အားနည်းချက် (Vulnerability) ရှိနေပါက၊ Hacker သည် ထိုအားနည်းချက်မှတစ်ဆင့် ကျူးကျော်ဝင်ရောက်နိုင်သည်။

### ကောက်ချက်
Firewall သည် ကွန်ယက်လုံခြုံရေး၏ အရေးကြီးသော ပထမဆုံး ကာကွယ်မှုအလွှာ (First Line of Defense) ဖြစ်သည်။ ၎င်းသည် ကွန်ယက်အသွားအလာကို စည်းမျဉ်းသတ်မှတ် စစ်ထုတ်ပေးခြင်းဖြင့် မလိုလားအပ်သော ဆက်သွယ်မှုများကို တားဆီးကာ ကျန်ခဲ့သော လုံခြုံရေး နည်းလမ်းများ (ဥပမာ- Antivirus, Intrusion Detection Systems, User Training) နှင့် အတူ အလုပ်လုပ်ရန် ဒီဇိုင်းထုတ်ထားသည်။ Cybersecurity ကို စတင်လေ့လာသူအနေနှင့် Firewall ၏ အခြေခံ လုပ်ဆောင်ချက်၊ အမျိုးအစားခွဲခြားပုံနှင့် ၎င်း၏ အခန်းကဏ္ဍကို နားလည်ထားခြင်းသည် အလွန်အခြေခံကျပြီး အရေးကြီးသော ဗဟုသုတတစ်ခု ဖြစ်ပါသည်။
